The request-response cycle

This section lists the events that transpire when a request arrives at Carrot's index.php. Here they are laid out chronologically:

System bootstrap

Carr

  • Requirements are checked. Execution stops when magic quotes or register globals are turned on.
  • Autoloading is set. Autoloader is instantiated and autoload.php is loaded.
  • DependencyInjectionContainer (DIC) is instantiated and providers.php is loaded.
  • Errors are turned into ErrorExceptions and exception handler is set.
  • FrontController is instantiated using DIC. An implementation of RouterInterface is injected at instantiation.
  • FrontController::dispatch() is called.

Routing the request

Before the FrontController

At the end, Carrot\Core\Interfaces\ResponseInterface::send() is called, sending the response header and body to the client.
