Carrot requires you to use its dependency injection container

You can't use Carrot without utilizing its dependency injection container. Carrot uses the container to instantiate your class and wire its dependencies, so without it, there's no Carrot. Using dependency injection container right at the core is what allows you to use plain old PHP objects for all your classes. There are of course drawbacks for this decision, but I think the benefits far outweigh them.
