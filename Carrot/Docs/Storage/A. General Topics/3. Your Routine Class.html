Your Routine Class

rou•tine, noun. a sequence of actions regularly followed; a fixed program. - New Oxford American Dictionary

When you write a web application, you want a subroutine to be run when a particular request is sent to your server. This routine code can be written inside a function, a method, or an anonymous function. For example, in Silex we write the routines inside anonymous functions, in Limonade we write them in regular functions, and in Kohana we write them in a controller class method.

Carrot is an object oriented framework, so your routine codes must be written inside a method. In Carrot, the methods that contain these routines is called routine methods . The classes/objects that contains these routine methods are called routine classes/objects.

The name routine class is chosen to be pattern independent. Unlike other frameworks, this class doesn't have to be your controller. Carrot is not an MVC framework and doesn't pretend to be one. You can implement MVC on top of Carrot, but it's not the only way you can use it. The only responsibility that you have to the framework is that your routine method returns an instance of Carrot\Core\Response (a value object) when called by the front controller. How exactly you plan on doing that is up to you.

You write your routine class as plain old PHP object, courtesy of the dependency injection container. Here's an example of a routine class: 

namespace Sample;

use Carrot\Core\Response;

class Welcome
{
    public function helloWorld()
    {
        return new Response("<h1>Hello World!<h1>");
    }
}
